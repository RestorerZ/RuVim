В этом каталоге находятся командные файлы программы Vim, используемые для
подсветки синтаксиса.

Указанные ниже файлы не предназначены для каких-либо языков, а требуются для
работы самого редактором Vim:

syntax.vim	применяется при обработке команды ":syntax on". Этот файл
		вызывает командный файл synload.vim.

manual.vim	применяется при обработке команды ":syntax manual". Этот файл
		вызывает командный файл synload.vim.

synload.vim	содержит автокоманды на загрузку файла с правилами синтаксиса
		для какого-то определённого языка, когда он был распознан по
		наименованию (расширению) файла. Также в этом файле содержатся
		команды для выполнения настройки меню «Синтаксис» в графическом
		интерфейсе.

nosyntax.vim	применяется при обработке команды ":syntax off". Этот файл
		отменяет загрузку файла synload.vim


Несколько специальных файлов:

2html.vim      — для преобразования любого файла, в котором включена подсветка
		 синтаксиса, в HTML-файл (только в графическом интерфейсе).
colortest.vim  — для сверки соответствия между наименованием цвета и цветом
		 на экране.
hitest.vim     — для просмотра текущих установок подсветки синтаксиса.
whitespace.vim — для подсветки символов табуляции и пробелов.


Если планируете создать файл с правилами подсветки синтаксиса для какого-то
языка программирования, то ознакомьтесь с документацией. Наберите для этого
команду ":help usr_44.txt".

Если вы создали файл с правилами подсветки синтаксиса и это может быть полезно
другим людям, отправьте, пожалуйста, этот файл на адрес электронной почты
Bram@vim.org. Опишите, как выполняется распознавание типов файлов для этого
языка, либо по расширению наименования файла, либо через проверку нескольких
строк в файле. При написании файла правил подсветки синтаксиса руководствуйтесь
рекомендациями из встроенной документации, подробнее см.  ":help 44.12".

В случае имеющихся замечаний по уже существующему файлу правил подсветки
синтаксиса, эти замечания лучше отправлять сопровождающему данный файл
специалисту. Способ связи с этим специалистом, обычно, указывается в заголовке
самого файла. И только в случае, когда не получен ответ от этого специалиста,
можно отправить свои замечания по адресу bram@vim.org.

В свою очередь, если вы является специалистом, занимающийся поддержкой какого-то
файла правил синтаксиса и вносите исправления или дополнения в этот файл, то не
забывайте отправлять новые версии файла по адресу bram@vim.org.

Дополнительную информацию см. в документации редактора Vim, ":help syntax".
